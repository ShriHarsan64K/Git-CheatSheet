Git Cheatsheet

# every command you actually use, with a reason why

Setup & Config

$git config --global user.name "Your Name"

Set your name — shown on every commit you make

$git config --global user.email "[email protected]"

Set your email — must match your GitHub account for contributions to count

$git config --global core.editor vim

Set default editor for commit messages (vim / nano / code)

$git config --global init.defaultBranch main

Make all new repos start on main instead of master

$git config --list

Show all current git configuration values

$git init

Initialise a brand-new local repo in the current folder

Basic Workflow

$git status

Show which files are staged, unstaged, or untracked

$git add <file>

Stage a specific file for the next commit

$git add .

Stage every changed and new file in the current directory

$git commit -m "message"

Save staged changes with a description message

$git commit --amend

Edit the last commit message or add forgotten files to it

$git diff

Show line-by-line changes not yet staged

$git diff --staged

Show changes that are staged and ready to commit

$git rm <file>

Delete a file and stage its removal in one step

Branching & Merging

$git branch

List all local branches — the active one is marked with *

$git branch <name>

Create a new branch without switching to it

$git checkout -b <name>

Create a new branch and switch to it immediately

$git switch <name>

Switch to an existing branch (modern alternative to checkout)

$git merge <branch>

Merge another branch into the current one

$git rebase <branch>

Replay your commits on top of another branch for a clean history

$git branch -d <name>

Delete a branch that has already been merged

$git branch -D <name>

Force-delete a branch even if it has unmerged commits

Remote / GitHub

$git clone <url>

Download a repo and its full history to your machine

$git remote -v

List all remote connections and their URLs

$git remote add origin <url>

Connect your local repo to a remote for the first time

$git push origin <branch>

Upload your local commits to the remote branch

$git push -u origin main

Push and set upstream so future git push needs no arguments

$git pull

Fetch remote changes and merge them into your current branch

$git fetch

Download remote changes without merging them — safe to inspect first

$git push --delete origin <branch>

Delete a branch on the remote (GitHub)

Undo & Fix

$git restore <file>

Discard unstaged changes in a file — reverts to last commit

$git restore --staged <file>

Unstage a file without losing your changes

$git reset HEAD~1 --mixed

Undo the last commit but keep the changes in your working directory

$git reset HEAD~1 --hard

Completely delete the last commit and all its changes — irreversible

$git revert <commit>

Create a new commit that reverses a specific commit — safe for shared branches

$git clean -fd

Delete all untracked files and directories — cannot be undone

Stash

$git stash

Temporarily save uncommitted changes so you can switch branches cleanly

$git stash pop

Restore the most recently stashed changes and remove them from the stash

$git stash list

Show all saved stashes with their index numbers

$git stash drop

Delete the most recent stash without applying it

Inspect & Log

$git log --oneline --graph

Compact visual history showing branches and merges

$git log -p <file>

Show the full change history of a specific file

$git show <commit>

Display the full details and diff of a specific commit

$git blame <file>

Show who last changed each line of a file and in which commit

$git shortlog -sn

Show commit counts per contributor — ranked by most commits

$git tag <v1.0>

Mark a specific commit as a release or milestone with a label

Advanced

$git cherry-pick <commit>

Apply a specific commit from another branch onto the current one

$git cherry-pick <A>..<B>

Apply a range of commits from A to B onto the current branch

$git cherry-pick --no-commit <commit>

Apply changes from a commit but don't auto-commit — lets you review first

$git bisect start

Begin a binary search through commits to find which one introduced a bug

$git bisect good <commit>

Mark a commit as working during a bisect session

$git bisect bad <commit>

Mark a commit as broken during a bisect session

$git bisect reset

End the bisect session and return HEAD to its original position

$git reflog

Show every HEAD movement — recover lost commits after a bad reset

$git reflog <branch>

Show the movement history of a specific branch — not just HEAD

$git submodule add <url>

Embed another git repo inside your repo as a tracked dependency

$git submodule update --init --recursive

Pull and initialise all submodules after cloning a repo that has them

$git archive --format=zip HEAD > out.zip

Export the current snapshot as a zip file — no git history included

$git notes add -m "note" <commit>

Attach a note to a commit without changing its hash

$git gc

Run garbage collection to compress history and clean up loose objects

$git fsck

Verify the integrity of the git object database — checks for corruption

Rebase

$git rebase <branch>

Replay your commits on top of another branch — creates a linear, cleaner history

$git rebase -i HEAD~3

Interactive rebase — rewrite, squash, reorder, or drop the last 3 commits

$git rebase -i origin/main

Interactively rebase all commits not yet on main — clean up before a PR

$git rebase --onto main old-base feature

Move feature branch from old-base onto main — precise branch transplanting

$git rebase --continue

Resume rebase after resolving a merge conflict

$git rebase --skip

Skip the current conflicting commit and continue the rebase

$git rebase --abort

Cancel the rebase entirely and return to the state before it started

$git rebase --autosquash -i origin/main

Auto-arrange fixup! and squash! commits during interactive rebase

$git rebase --autostash <branch>

Auto-stash uncommitted changes before rebase and restore them after

$git commit --fixup=<commit>

Create a fixup commit that auto-squashes into a specific earlier commit on rebase

$git pull --rebase

Pull remote changes and rebase your local commits on top instead of merging

$git config --global pull.rebase true

Make rebase the default behaviour for all future git pulls

Worktree

$git worktree add <path> <branch>

Check out a branch into a separate folder — work on two branches simultaneously without stashing

$git worktree list

Show all linked worktrees and their current branches

$git worktree remove <path>

Remove a linked worktree that is no longer needed

$git worktree prune

Clean up stale worktree references — runs automatically with gc

Hooks & Automation

$ls .git/hooks/

List all available hook scripts — rename any .sample file to activate it

$chmod +x .git/hooks/pre-commit

Make a hook executable — hooks must be executable or git ignores them

$git commit --no-verify

Skip pre-commit and commit-msg hooks — useful when hooks are blocking a valid commit

$git config --global core.hooksPath ~/.githooks

Set a global hooks directory — share hooks across all repos without copying them

Config & Aliases

$git config --global alias.st status

Create a global alias — git st now runs git status

$git config --global alias.lg "log --oneline --graph --decorate"

Create an alias for a complex command — git lg shows a pretty log

$git config --global core.autocrlf input

Convert CRLF to LF on commit — essential on Linux/macOS to avoid line ending issues

$git config --global core.excludesfile ~/.gitignore_global

Set a global gitignore file that applies to every repo on your machine

$git config --global merge.tool vimdiff

Set your preferred merge conflict resolution tool

$git config --global diff.tool vscode

Use VS Code as your diff viewer for git difftool

$git config --local user.email "work@company.com"

Override global config for a single repo — useful for work vs personal projects

$git config --global rerere.enabled true

Remember how you resolved a conflict and auto-apply it next time the same conflict appears
