Git Push

# every push variant — what it does and when to use it

Golden rule: never force-push to a shared / main branch. Force push only on your own feature branches that no one else has checked out.

The Basics

$ git push

Push current branch to its tracked upstream — works once upstream is set

$ git push origin main

Push the local main branch to the remote named origin

$ git push -u origin main

Push and set upstream — future git push needs no arguments on this branch

$ git push origin feature-xyz

Push a local feature branch to remote for the first time

$ git push origin local:remote

Push a local branch to a remote branch with a different name

$ git push --all origin

Push every local branch to the remote in one go

Branch Management

$ git push origin --delete feature-xyz

Delete a branch on the remote — useful after a PR is merged

$ git push origin :feature-xyz

Shorthand to delete a remote branch — the colon means "push nothing to it"

$ git push --set-upstream origin <branch>

Link a local branch to a remote branch so tracking is set up

$ git push origin HEAD

Push the current branch without typing its name — HEAD always refers to where you are

Tags & Releases

$ git push origin v1.0.0

Push a specific tag to remote — tags are not pushed automatically

$ git push --tags

Push all local tags to the remote at once

$ git push --follow-tags

Push commits and any annotated tags that point to those commits — recommended over --tags

$ git push origin --delete v1.0.0

Delete a tag from the remote — does not delete the local tag

Force Push

$ git push --force

Overwrite remote history with local history — overwrites others' work if used on a shared branch

$ git push --force-with-lease

Safer force push — refuses if someone else pushed since your last fetch, preventing accidental overwrites

$ git push --force-with-lease --force-if-includes

Even safer — also checks that remote-tracking refs are up to date before allowing the force push

Useful Options

$ git push --dry-run

Simulate the push and show what would happen — nothing is actually sent

$ git push --verbose

Show detailed output of what is being pushed and to where

$ git push --no-verify

Skip pre-push hooks — useful when hooks are failing but you know the push is safe

$ git push --atomic

Push multiple refs as one atomic operation — all succeed or all fail together

$ git push --mirror

Mirror all refs including branches, tags, and deletions — used for full repo backups

$ git push --prune

Remove remote branches that no longer have a local counterpart

Common Patterns

$ git push origin HEAD --force-with-lease

The safest way to force-push the current branch after a rebase or amend

$ git push -u origin HEAD

Push current branch and set upstream in one command — works on any branch without typing its name

$ git push origin --all --follow-tags

Push all branches and all their associated tags in one command — great for initial remote setup

$ git push origin --delete feature && git branch -d feature

Delete a branch both on remote and locally in one line — full cleanup after merging a PR
